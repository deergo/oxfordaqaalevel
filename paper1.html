1. Procedural Programming

Data Types

Integer, real/float, Boolean, character, string, date/time. Know typical operations and storage implications.

Programming Concepts

  • Variables and assignment
  • Selection (if / elif / else or switch)
  • Iteration (definite and indefinite)
  • User input/output
  • Meaningful identifiers and comments

Arithmetic / Relational / Boolean Operations

  • Arithmetic: +, −, ×, ÷ (real), DIV, MOD, exponentiation; rounding vs truncation
  • Relational: =, ≠, <, >, ≤, ≥
  • Boolean: NOT, AND, OR, XOR (precedence: NOT > AND > OR)

Strings & Conversions

  • Length, position/index, substring, concatenation
  • Char/code conversions
  • Type conversions between string, integer, float, date

Random Numbers

Generate random integers/reals for simulations, testing, and sampling.

Exception Handling

Use try/catch patterns to manage run-time errors and ensure graceful recovery.

Subroutines & Scope

  • Parameters (by value/reference), return values
  • Local vs global variables
  • Modularisation benefits; stack frames (return address, params, locals)

2. Fundamental Data Structures

Arrays & Lists

1D/2D arrays or dynamic lists; understand static vs dynamic storage.

Records

Group related fields; equivalents include Python dataclass or C# struct.

Queues

  • FIFO; enqueue and dequeue
  • Linear and circular queue implementations

Stacks

  • LIFO; push, pop, peek/top
  • Array-based or library implementations

3. Program Design

Structured Programming

Modules, parameters, scope; hierarchy and structure charts.

Abstraction & Decomposition

Abstraction removes unnecessary detail; decomposition breaks problems into sub-tasks.

Algorithms & Pseudocode

Design, trace, test, and convert pseudocode to code; algorithms must be finite and effective.

Software Development Stages

  • Analysis → Design → Implementation → Testing → Evaluation
  • Test with normal, boundary, and erroneous data

4. Searching and Sorting Algorithms

Searching

  • Linear search: sequential scan
  • Binary search: divide-and-conquer; requires sorted data
  • Compare conceptual efficiency (time) at a high level

Sorting

  • Bubble sort: iterative adjacent swaps
  • Merge sort: divide, sort, merge
  • Discuss suitability, efficiency, and memory footprint
