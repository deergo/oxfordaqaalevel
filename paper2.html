6. Computer Systems

Hardware & Software

What is Hardware?

Exam-style definition: Hardware refers to the physical components of a computer system ‚Äî the parts that can be touched ‚Äî that work together to process, store, and communicate data.

Examples:

  • Input devices: keyboard, microphone, scanner
  • Output devices: monitor, printer, speakers
  • Storage devices: SSD, hard drive, USB flash drive
  • Processing devices: CPU, GPU
  • Communication devices: network interface cards, routers

Exam tip: If asked, ‚Äúname a component on a sound card that transforms a continuous signal into a form that can be stored by a computer,‚Äù the answer is Analogue to Digital Converter (ADC).

Software Overview

Exam-style definition: Software is a set of instructions or programs that tell the hardware what to do. It is intangible and enables a computer to perform useful tasks.

Key distinction: Hardware performs operations; software provides the instructions that control it.

Exam pattern: ‚ÄúState the difference between hardware and software.‚Äù One-mark answer: ‚ÄúHardware is the physical equipment; software is the set of programs that control it.‚Äù

Firmware

Firmware is software stored permanently in ROM. It provides low-level control of hardware devices ‚Äî for example, the BIOS that initialises hardware when a computer starts.

Relationship Between Hardware and Software

Hardware RoleSoftware Role
Executes machine code instructionsProvides those instructions
Performs physical operationsDefines logical processes
Dependent on software to functionDependent on hardware to execute

Example: The CPU executes instructions provided by the Operating System (software), which in turn controls input/output hardware such as printers or screens.

System and Application Software

What is System Software?

Exam-style definition: System software enables the hardware and user applications to communicate. It manages hardware operations, provides an interface for the user, and ensures efficient execution of programs.

Utility Software

Utilities are system programs designed to perform maintenance or optimisation tasks.

  • Examples: file compression, virus scanning, defragmentation, system monitoring, backup and recovery

Exam tip: mark schemes often award one mark per correct example of a utility.

Device Drivers

Definition: Programs that allow the operating system to communicate with hardware peripherals.

Example: A printer driver translates generic OS instructions into device-specific commands.

Why drivers are needed: the OS requires a driver to translate its commands into the form the hardware understands.

Application Software

Exam-style definition: Application software is written to perform specific user-oriented tasks, rather than managing hardware.

  • General-purpose: e.g., word processors, spreadsheets
  • Special-purpose: e.g., payroll system
  • Bespoke: developed for a single organisation

  • Explain why a driver is needed for a new device.

Operating System (OS) ‚Äì Functions

Purpose: The OS acts as an interface between hardware and software. It hides hardware complexity from users and applications.

  1. Resource Management ‚Äì controls CPU time, memory, and peripherals
  2. File Management ‚Äì stores, retrieves, and organises files
  3. Process Management ‚Äì multitasking and scheduling
  4. Security Management ‚Äì access rights and permissions
  5. User Interface ‚Äì GUI and/or CLI
  6. Device Management ‚Äì communicates with hardware via device drivers

Exam tip: a valid ‚Äúone function of an OS‚Äù answer could be: managing memory or providing a user interface.

Programming Languages & Translation

Software that converts source code into machine code is called a translator.

TypeFunctionExample Use
AssemblerConverts assembly language to machine codeLow-level hardware control
CompilerTranslates high-level code into an executable filePerformance and distribution (e.g., C#, Java)
InterpreterTranslates and executes code line-by-lineScripting and rapid testing (e.g., Python)
  • Know the distinction between source and object code.
  • Some languages target bytecode for execution on a virtual machine.

Exam link: assembly language requires an assembler; high-level languages are compiled or interpreted.
